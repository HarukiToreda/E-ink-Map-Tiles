# InkHUD Firmware Builder

Standalone Windows tool for building and flashing Meshtastic InkHUD firmware. No local development environment required.

> **Companion tool** — works alongside [E-ink Map Tiles](README.md), which generates the `MapTile.h` file that this tool bakes into firmware.

## What It Does

- Downloads a portable Python environment and installs PlatformIO locally on first run (progress shown in a live log pane).
- Clones the [meshtastic/firmware](https://github.com/meshtastic/firmware) repository and keeps it up to date with a shallow fetch before every build.
- Optionally copies a `MapTile.h` generated by the EinkMapTile Tool into the firmware repo before building.
- Builds firmware for any of the 14 supported InkHUD devices via PlatformIO.
- Flashes built firmware directly to a connected device over USB serial using esptool.
- Uploads any previously built `.bin` file to a device without rebuilding.

No Git, Python, PlatformIO, or VS Code installation is required on the target machine.

## Basic Flow

1. Run `InkHUDBuilder.exe`.
2. On first launch, a setup dialog downloads the portable Python environment and installs PlatformIO (one-time, ~5 min). Progress is shown line by line in the log pane.
3. Select a **Target device** from the dropdown (devices are grouped by chip family).
4. Optionally set a **MapTile.h** path — use **⬡ Export for InkHUD** in the EinkMapTile Tool to generate this file, then paste the path here. Leave blank to build firmware without custom map tiles.
5. Select a **firmware branch**: `develop` (latest, default) or `master` (stable release).
6. Click **Build** to compile. The first build per chip family downloads the toolchain (~300–500 MB) and compiles the full IDF framework — this takes 5–15 minutes. Subsequent builds are fast (incremental).
7. To also flash immediately after building, check **Flash after build** and select a COM port before clicking **Build**.
8. To flash a previously built `.bin` without rebuilding, select a COM port and click **Upload**, then pick the `.bin` file from the file picker.

Built firmware files are saved alongside the `MapTile.h` file (or in `%APPDATA%\InkHUDBuilder\firmware_builds\` if no MapTile.h is set).

## MapTile.h

The **MapTile.h** field is optional. If left blank, the firmware is built using whatever map header is already in the cloned repo (or none if this is a clean checkout). Fill it in to bake custom offline map tiles into the firmware image.

The hint text reads: *Generate MapTile.h file from EinkMapTile Tool* — use the **⬡ Export for InkHUD** button in that tool to produce the file, then point the builder at it.

## COM Port and Flashing

The COM port dropdown shows the device name reported by the USB driver alongside the port number (e.g. `COM4 — USB-SERIAL CH340`). Click **↻** to refresh the list after plugging in a device.

**Flash after build** — when checked, the build and flash happen in a single PlatformIO invocation. No rebuild occurs between compile and flash.

**Upload** — opens a file picker pointing at the firmware output folder. Select any `.bin` to flash it directly via esptool. If you pick a regular app `.bin`, the tool automatically locates the matching `.factory.bin` (merged bootloader + partition table + app) and flashes that at address `0x0` instead — the same approach used by the Meshtastic web flasher.

nRF52840 devices use UF2 drag-and-drop flashing. The Upload button shows instructions and the `.uf2` file path when an nRF52840 target is selected.

## Firmware Branch

- **develop** — the main development branch, tracking the latest Meshtastic firmware commits. Use this for the newest InkHUD features.
- **master** — the stable release branch.

Switching branches triggers a re-clone if the cached repo is on the wrong branch.

## Clean / Reset

The **Clean** button (in the top panel) removes the entire local cache under `%APPDATA%\InkHUDBuilder\` — the portable Python environment, PlatformIO installation, cloned firmware repo, and all build artifacts. Restart the app after cleaning to re-run first-time setup from scratch.

## Building from Source

`InkHUDBuilder.exe` is built from `scripts/build_inkhud_gui.py` using PyInstaller:

```bash
pip install pyinstaller pyserial
pyinstaller InkHUDBuilder.spec --noconfirm
# Output: dist/InkHUDBuilder.exe
```
